## Why Prompt Master

Every AI user wastes credits the same way: write a vague prompt, get the wrong output, re-prompt, get closer, re-prompt again, and finally land the right answer on attempt four. That's three wasted API calls — multiplied by 50 prompts a day, that's real money and real time gone.

Prompt Master fixes this by treating every prompt as an engineering problem. It knows which AI tool you're targeting, what format that tool responds to best, and which words in your request are load-bearing versus noise. The result is a single prompt that replaces the re-prompting loop.

## Quick Example

Tell Claude naturally what you need — no special syntax required:

```
Write me a prompt for Cursor to refactor my auth module
```

```
I need a prompt for Claude Code to build a REST API — ask me what you need to know
```

```
Here's a bad prompt I wrote for GPT-4o, fix it: [paste prompt]
```

Or invoke explicitly with the skill name:

```
/prompt-master

I want to ask Claude Code to build a todo app with React and Supabase
```

<Note>
Prompt Master activates **only** when you explicitly ask it to write, fix, improve, or adapt a prompt for a specific AI tool. It does not interfere with regular Claude conversations, coding tasks, or document writing.
</Note>

## Get Started

<Steps>
  <Step title="Install the skill">
    Download the repo as a ZIP and upload it to **claude.ai → Customize → Skills**. See [Installation](/installation) for full steps.
  </Step>
  <Step title="Invoke it naturally">
    Ask Claude to write a prompt for any AI tool. Prompt Master detects the target tool and routes automatically.
  </Step>
  <Step title="Copy and paste">
    Receive a single copyable prompt block with a one-line strategy note. Paste it directly into your target tool.
  </Step>
  <Step title="First-attempt success">
    The prompt is already optimized for the target tool's syntax, token budget, and known failure modes. No re-prompting needed.
  </Step>
</Steps>
